# Issue Review

## 대상 Branch 선정 방법
- 인자가 하나 있거나, 없을 수 있다.
- 인자가 있는 경우 이름이 $ARG1인 Branch를 대상으로 한다.
- 인자가 없는 경우 현재 Branch를 대상으로 한다.

## 작업 순서
1. Branch 이름에서 JIRA 이슈 번호를 추출한다.
2. Branch 범위를 계산한다.
3. Branch 범위의 작업 내용을 분석한다.
4. 분석한 작업 내용을 JIRA 이슈의 댓글로 생성한다.

## 이슈 댓글 작성 형식
반드시 첫 줄은 🤖 Claude Code 🤖 를 작성하고, 빈 줄을 하나 추가한 후에 분석한 작업 내용을 작성한다.  
내용은 각각의 작업 목록을 나열하고, 각 작업에 대해 분석한 내용을 작성한다.

## 분석 시 고려 사항
- 작업 분석 시 Git의 Commit Log를 활용한다.  
- 해당 작업의 효과와 영향 범위, 잠재적 위험 및 오류를 중점적으로 분석한다.  
- 작업 범위를 벗어난 부분은 철저히 제외한다.  
- 새로운 지식을 알려주는 것이 아닌 실제로 일어난 일을 정확히 분석하고 보고하는 것을 목적으로 한다.
