# Code Review

## Code Review 대상
- 인자는 0~1개이다.
- 인자가 있는 경우 이름이 $ARG1인 Branch를 대상으로 한다.
  - 이름이 $ARG1과 완벽히 같은 Branch가 없으면 Branch가 없음을 알리고 중단한다.
- 인자가 없는 경우 현재 Branch를 대상으로 한다.

## Code Review 시 고려사항
- 버그, 보안, 성능, MVVM 패턴 준수 여부, 코드 품질, 메모리 누수 관점에서 리뷰하고 문제점에 대해 자세히 리뷰한다.
- 파일 전체가 아닌 수정 한 내용만 리뷰한다.

## Agent 및 Skill 활용
- 각 작업에 적합한 Agent, Skill이 있으면 사용한다.

## 리뷰 결과
- 브랜치 정보를 가장 처음에 표시한다.
- 문제점 표시 할 때는 문제점을 발견 한 파일과 줄 번호, 해결 방법을 항상 표시한다.
